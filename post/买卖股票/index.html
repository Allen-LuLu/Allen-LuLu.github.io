一、买卖股票的最佳时机（限定买卖次数为1次）

  • 问题：121. 买卖股票的最佳时机

  • 分析

    • 动规五部曲

      • 分析dp数组的含义：在股票问题中，首先要根据题目分析题目会有几种状态！
        • 本题中，股票一共有两种状态
          • 持有股票
          • 不持有股票
        • 故，本题的dp数组对于每个i有两个状态
          • dp[i][0]：在第i天仍持有股票时所能达到的最大利润
          • dp[i][1]：在第i天不持有股票时能达到的最大利润
        • 注意：第i天持有股票，不一定是在第i天当天买的股票，有可能是前几天买的，但没卖！同理，第i天不持有股票，也不一定是第i天当天卖出去的股票，有可能是前几天就卖了！
      • 推导公式
        • dp[i][0]:第i天持有股票，共有两种可能
          • 第i-1天就已经持有股票了，一直没卖，dp[i][0]=dp[i-1][0]
          • 第i-1天才买入股票，由于本题只允许买卖一次，因此第i天才买入股票，说明前面几天都没有任何操作，因此利润为0，此时买入股票，利润变为负数，dp[i][0]=-prices[i]
          • 综上，dp[i][0]=max(dp[i-1][0],-prices[i])
        • dp[i][1]:第i天不持有股票，同样有两种可能
          • 第i-1天就没有股票在手，股票要么没买，要么早就卖掉了。dp[i][1]=dp[i-1][1]
          • 第i-1天还有股票，第i天给它卖了，此时,dp[i][1]=dp[i-1][0]+prices[i]
          • 综上，dp[i][1]=max(dp[i-1][1],dp[i-1][0]+prices[i])
        • 综上
          • dp[i][0]=max(dp[i-1][0],-prices[i])
          • dp[i][1]=max(dp[i-1][1],dp[i-1][0]+prices[i])
      • 初始化：由递推公式可知，我们需要初始化dp[0][0]和dp[0][1]
        • 如果第0天我们持有股票，说明我们在第0天就买入了，此时，dp[0][0]=-prices[0]
        • 如果第0天我们不持有股票，说明我们什么操作都没有，此时，dp[0][1]=0
      • 遍历顺序：由递推公式可知，从前往后遍历
      • 结果，我们要的肯定是最后一天不持有股票时的最大利润！（因为最后一天你还拿着股票你又卖不出去，钱烂在手里了）

      func maxProfit(prices []int) int {
    dp := make([][2]int,len(prices))
    dp[0][0],dp[0][1] = -prices[0],0

    for i:=1;i<len(dp);i++{
        dp[i][0] = max(dp[i-1][0],-prices[i])
        dp[i][i] = max(dp[i-1][1],dp[i-1][0]+prices[i])
    }
    return dp[len(prices)-1][1]
}

func max(a,b int)int{
    if a > b{
        return a
    }
    return b
}

二、买卖股票的最佳时机II（不限定买卖次数）

  • 问题：122. 买卖股票的最佳时机 II

  • 分析

    • dp数组的意义：分析股票的状态！，本题股票允许被多次买卖，但总的来说状态还是只有两种

      • dp[i][0]:第i天仍持有股票时获得的最大利润
      • dp[i][1]:第i天不持有股票时获得的最大利润

    • 推导

      • dp[i][0]:第i天持有股票，分两种可能
        • 第i-1天就不持有股票，dp[i][0]=dp[i-1][0]
        • 第i天才买入股票，由于本题允许多次买卖，因此之前可能已经发生过交易，故之前的利润不一定为0，这就是本题与上一题的差别所在！因此，dp[i][0]=dp[i-1][1]-prices[i]
        • 综上，dp[i][0]=max(dp[i-1][0],dp[i-1][1]-prices[i])
      • dp[i][1]:第i天不持有股票，同样有两种可能
        • 第i-1天就没有股票在手，股票要么没买，要么早就卖掉了。dp[i][1]=dp[i-1][1]
        • 第i-1天还有股票，第i天给它卖了，此时,dp[i][1]=dp[i-1][0]+prices[i]
        • 综上，dp[i][1]=max(dp[i-1][1],dp[i-1][0]+prices[i])
      • 综上
        • dp[i][0]=max(dp[i-1][0],dp[i-1][1]-prices[i])
        • dp[i][1]=max(dp[i-1][1],dp[i-1][0]+prices[i])

    • 初始化：与之前一致

    • 遍历顺序：从前往后

    • 进阶：观察递推公式可以发现，第[i]行的数据完全由第[i-1]行决定，因此，可以进行维度压缩，将第i-1行数据复制到第i行使用，可以节省大量空间！

    func maxProfit(prices []int) int {
    var dp [2]int
    dp[0],dp[1] = -prices[0],0

    for i:=1;i<len(prices);i++{
        dp[0] = max(dp[0],dp[1]-prices[i])
        dp[1] = max(dp[1],dp[0]+prices[i])
    }
    return dp[1]
}

三、买卖股票的最佳时机III（限定买卖次数最多为2次）

  • 问题：123. 买卖股票的最佳时机 III

  • 分析

    • dp数组的含义：分析本题状态，共有以下五种状态！
      • 0：还未买入，不操作
      • 1：第一次持有
      • 2：第一次不持有
      • 3：第二次持有
      • 4：第二次不持有
    • 推导
      • dp[i][0]:第i天还未进行任意一次买入，不操作，说明前面都未操作，dp[i][0]=dp[i-1][0]
      • dp[i][1]:第i天第一次持有，有以下两种可能
        • 第i-1天就第一次持有股票了，dp[i][1]=dp[i-1][1]
        • 第i天才第一次买入股票，dp[i][1]=dp[i-1][0]-prices[i]
      • dp[i][2]:第i天第一次不持有股票，有以下两种可能
        • 第i-1天就第一次不持有股票了，即第一次股票出售早就发生了，此时，dp[i][2]=dp[i-1][2]
        • 第i天第一次出售股票，即第i-1天仍然属于第一次持有股票,dp[i-1][2]=dp[i-1][1]+price[i]
      • dp[i][3]:第i天第二次持有股票，有以下两种可能
        • 第i-1天就第二次持有股票了，即第二次股票购买早就发生了，此时，dp[i][3]=dp[i-1][3]
        • 第i天第二次购买股票，即第i-1天属于第一次不持有股票的状态,dp[i-1][3]=dp[i-1][2]-price[i]
      • dp[i][4]:第i天第二次不持有股票，有以下两种可能
        • 第i-1天就第二次不持有股票了，即第二次股票出售早就发生了，此时，dp[i][4]=dp[i-1][4]
        • 第i天第二次出售股票，即第i-1天仍然属于第二次持有股票,dp[i-1][4]=dp[i-1][3]+price[i]
      • 综上
        • dp[i][0]=dp[i-1][0]
        • dp[i][1]=max(dp[i-1][1],dp[i][1]=dp[i-1][0]-prices[i])
        • dp[i][2]=max(dp[i-1][2],dp[i][1]=dp[i-1][1]+prices[i])
        • dp[i][3]=max(dp[i-1][3],dp[i][1]=dp[i-1][2]-prices[i])
        • dp[i][4]=max(dp[i-1][4],dp[i][1]=dp[i-1][3]+prices[i])
    • 初始化
      • dp[i][0]=0
      • dp[i][1]=-prices[0]
      • dp[i][2]=0
      • dp[i][3]=-prices[0]
      • dp[i][4]=0
    • 遍历顺序：从前往后
    • 进阶：将第i-1行拷贝到第i行，进行维度压缩

    func maxProfit(prices []int) int {
    var dp [5]int
    dp[0],dp[1],dp[2],dp[3],dp[4]=0,-prices[0],0,-prices[0],0

    for i:=0;i<len(prices);i++{
        dp[0] = dp[0]
        dp[1] = max(dp[1],dp[0]-prices[i])
        dp[2] = max(dp[2],dp[1]+prices[i])
        dp[3] = max(dp[3],dp[2]-prices[i])
        dp[4] = max(dp[4],dp[3]+prices[i])
    }
    return dp[4]
}

    if a > b{
        return a
    }
    return b

四、买卖股票的最佳时机IV（限定买卖次数最多为k次）

  • 问题：188. 买卖股票的最佳时机 IV

  • 分析：

    • 当k取2时，和上一题没有任何区别，因此，思路就是上一题的思路，细心解题就可以发现上一题的规律
    • 总共有**2k+1**种状态！
    • 当i为奇数时，为持有股票状态，此时要么是前一天的，要么是当天买的
    • 当i为偶数时，为不持有股票的状态，此时要么利润是前一天，要么是当天卖的
    • 初始化时，奇数i的利润为-prices[0]

    func maxProfit(k int, prices []int) int {
    if len(prices) == 0 || k==0{
        return 0
    }
    dp := make([]int,2*k+1)
    for i:=1;i<len(dp);i=i+2{
        dp[i] = -prices[0]
    }

    for i:=1;i<len(prices);i++{
        for j:=1;j<=2*k;j++{
            if j%2 == 1{
                dp[j] = max(dp[j],dp[j-1]-prices[i])
            }else if j % 2 == 0{
                dp[j] = max(dp[j],dp[j-1]+prices[i])
            }
        }
    }
    return dp[2*k]
}

func max(a,b int)int{
    if a > b{
        return a
    }
    return b
}

五、最佳买卖股票时机含冷冻期（不限定交易次数）

  • 问题：309. 最佳买卖股票时机含冷冻期

  • 分析

    • dp数组的意义：本题共分为三种状态！但理解起来有点难度。在本题中，使用的是**第i天结束后的状态！**比如第i天结束后是冷冻期，那么第i+1天就不能购买股票！
      • dp[i][0]：第i天结束后仍然持有股票
      • dp[i][1]：第i天结束后未持有股票，且处于冷冻期
      • dp[i][2]：第i天结束后未持有股票，且未处于冷冻期
    • 推导
      • dp[i][0]：第i天结束后仍持有股票,有两种可能
        • 第i-1天就持有股票，即dp[i][0]=dp[i-1][0]
        • 第i天才买入的股票，要第i天买入股票，那么第i-1天结束后就不能持有股票，且不能是冷冻期（否则第i天不能买股票），因此，dp[i][0]=dp[i-1][2]-prices[i]
      • dp[i][1]:第i天结束后未持有股票，且处于冷冻期
        • 要使得第i天结束后进入冷冻期，那么就只能在第i天交易股票了，则第i-1天是持有股票的，即dp[i][1]=dp[i-1][0]+prices[i]
      • dp[i][2]:第i天结束后未持有股票，且不是冷冻期，有两种可能
        • 前一天就是这种状态，即dp[i][2]=dp[i-1][2]
        • 前一天结束后进入冷冻期，今天是冷冻期，今天结束后进入未持有股票，且未处于冷冻期状态，这种情况下今天也是没有任何操作的，即dp[i][2]=dp[i-1][1]
      • 综上
        • dp[i][0]=max(dp[i-1][0],dp[i-1][2]-prices[i])
        • dp[i][1]=dp[i-1][0]+prices[i]
        • dp[i][2]=max(dp[i-1][1],dp[i-1][2])
    • 初始化
      • dp[0][0]=-prices[0]:第一天就买入股票
      • dp[0][1]=0:不存在第一天就进入冷冻期的情况
      • dp[0][2]=0：第一天也卖不了股票
    • 遍历顺序：从前往后
    • 思考：能否维度压缩？
      • 乍一看，这题和前面几题一样，都是第i行数据只和第i-1行有关，立刻想到维度压缩，但这题一压缩就会出错！！！
      • 为什么？原因出在这里：dp[i][0]=max(dp[i-1][0],dp[i-1][2]-prices[i])
        • 前面的题之所以可以维度压缩，除了第i行数据只和第i-1行有关外，还有一个重要点在于，对于同一行，dp[i]也只由它前面的数据决定！
        • 在我们从前往后的遍历顺序中，前面的数据先被更新，后面的数据后被更新，如果后面的数据只与前面有关，这样就可以达到最终目的
        • 但本题中，dp[i][0]=max(dp[i-1][0],dp[i-1][2]-prices[i])，对于同一行数据，dp[0]不止与前面的dp[0]有关，还与后面的dp[2]有关，此时dp[2]还没被更新到，因此会出现错误

    func maxProfit(prices []int) int {
    dp := make([][4]int,len(prices))
    dp[0][0],dp[0][1],dp[0][2],dp[0][3] = -prices[0],0,0,0
    for i:=1;i<len(dp);i++{
        dp[i][0] = max(dp[i-1][0],max(dp[i-1][1]-prices[i],dp[i-1][3]-prices[i]))
        dp[i][1] = max(dp[i-1][1],dp[i-1][3])
        dp[i][2] = dp[i-1][0]+prices[i]
        dp[i][3] = dp[i-1][2]
    }
    return max(dp[len(prices)-1][3],max(dp[len(prices)-1][1],dp[len(prices)-1][2]))
}

func max(a,b int)int{
    if a > b{
        return a
    }
    return b
}

六、买卖股票的最佳时机含手续费（不限交易次数）

  • 问题：714. 买卖股票的最佳时机含手续费

    • 这道题与122. 买卖股票的最佳时机 II 几乎一模一样，就是成本变成股票价加上手续费罢了

    func maxProfit(prices []int, fee int) int {
    dp := make([][2]int,len(prices))
    dp[0][0],dp[0][1] = -prices[0]-fee,0
    for i:=1;i<len(dp);i++{
        dp[i][0]= max(dp[i-1][0],dp[i-1][1]-prices[i]-fee)
        dp[i][1] = max(dp[i-1][1],dp[i-1][0]+prices[i])
    }
    return dp[len(prices)-1][1]

}

总结

  • 1.股票问题，首要工作是根据题目分析总共有多少种股票状态！
  • 买入股票时要用此次买入前的利润减去成本（股价+可能存在的手续费）
    • 如果限定只能一次交易，那么没买入前的利润就是为0
    • 如果交易次数大于1次，此次买入前的利润就是上一次卖出时的利润
  • 卖出时要用当前持有股票的利润加上股价
  • 能否维度压缩，要看两个条件
    • 第i行数据只由第i-1行决定
    • 对于同一行数据，dp[i]只由其前面的数据有关，不能既与前面有关，又与后面有关！
